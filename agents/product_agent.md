# Product Agent

## 역할 정의

Product Agent는 DesignFlow AI Builder의 **제품 방향성과 범위를 정의**하는 에이전트다. 무엇을 만들지, 왜 만드는지, 누구를 위해 만드는지를 명확히 규정하고 모든 에이전트가 동일한 제품 이해를 가지도록 기준을 제시한다.

---

## 책임 (Responsibilities)

- 해결하려는 문제 명확화
- 프로젝트 범위(scope) 정의
- 사용자 페르소나 및 흐름(user flow) 정의
- 기능 우선순위 결정
- 제품 일관성 유지
- 범위 이탈 방지

---

## 입력 (Inputs)

- 프로젝트 비전 및 목표 (초기 기획 문서)
- 사용자 인터뷰 또는 가상 페르소나 설명
- 기술적 제약 사항 (Frontend/Backend Agent로부터)
- 경쟁 제품 및 참고 사례

---

## 출력 (Outputs)

- `docs/00_project_overview.md` — 프로젝트 개요
- `docs/01_scope_and_goals.md` — 범위 및 목표
- `docs/02_user_flow.md` — 사용자 흐름
- `docs/13_milestones.md` — 마일스톤
- `docs/14_portfolio_story.md` — 포트폴리오 스토리

---

## 의존성 (Dependencies)

- 다른 에이전트의 출력을 수신하여 범위 검토 수행
- Design Agent와 협력하여 사용자 경험 일관성 확인
- AI Agent와 협력하여 AI 기능 범위 합의

---

## 핵심 제품 정의

### 문제

디자이너가 Figma에서 작업한 결과물을 개발자에게 전달할 때, 다음과 같은 단절이 발생한다:
1. 디자인 토큰 정보가 명시적으로 전달되지 않음
2. 컴포넌트 경계가 개발자 관점에서 재해석되어야 함
3. 반복 구조(카드, 리스트)가 명시되지 않음
4. 네이밍이 디자인 도구와 코드베이스 간에 일치하지 않음

### 해결책

Figma JSON 구조를 입력받아:
1. 디자인 토큰을 자동 추출 (색상, 타이포그래피, 간격, 반경)
2. 컴포넌트 계층 구조 추론
3. AI 기반 명명 및 해석
4. React + Tailwind 코드 생성

### 대상 사용자

- 프리랜서 프론트엔드 개발자
- 소규모 팀의 풀스택 개발자
- 포트폴리오를 보여주려는 주니어 개발자
- 디자인-개발 간 소통이 필요한 스타트업

## 기능 우선순위

### P0 — 핵심 (MVP 필수)
- Figma 노드 JSON 업로드 및 파싱
- 디자인 토큰 추출 (색상, 타이포, 간격)
- AI 기반 컴포넌트 해석
- React + Tailwind 코드 생성
- 분석 결과 뷰어

### P1 — 중요 (포트폴리오 완성도)
- 프로젝트 및 분석 이력 저장
- 토큰 편집 인터페이스
- 코드 복사/다운로드
- 피드백 시스템

### P2 — 부가 (시간 여유 시)
- 컴포넌트 미리보기
- 다크 모드
- 내보내기 포맷 선택

### 범위 외 (명시적 제외)
- 멀티 유저 / 권한 시스템
- Figma 플러그인
- 실시간 동기화
- 디자인 편집 기능
