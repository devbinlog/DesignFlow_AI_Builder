# 02. 사용자 흐름

## 목적

이 문서는 DesignFlow AI Builder를 사용하는 핵심 사용자 흐름을 정의한다. 각 흐름은 사용자의 진입점부터 최종 결과물 획득까지의 단계를 명확히 기술한다.

---

## 핵심 결정 사항

### 주요 사용자 흐름 3가지

1. **신규 프로젝트 분석 흐름** — 가장 핵심 흐름
2. **분석 결과 탐색 흐름** — 결과 확인 및 수정
3. **코드 획득 흐름** — 최종 산출물 추출

---

## 흐름 1: 신규 프로젝트 분석

### 단계

```
1. 랜딩 페이지 진입
   → "새 프로젝트 시작" 버튼 클릭

2. 프로젝트 생성
   → 프로젝트명 입력
   → 설명 (선택)
   → 생성 확인

3. Figma JSON 업로드
   → Figma > 프레임 우클릭 > "Copy as JSON" 방법 안내
   → JSON 텍스트 붙여넣기 또는 파일 업로드
   → 유효성 검사 (Figma JSON 형식 확인)

4. 분석 실행
   → "분석 시작" 버튼 클릭
   → 진행 상태 표시:
      ├── [1/4] 노드 파싱 중...
      ├── [2/4] 디자인 토큰 추출 중...
      ├── [3/4] AI 해석 중... (가장 오래 걸림)
      └── [4/4] 코드 생성 중...

5. 결과 뷰어로 이동
   → 분석 결과 자동 표시
```

### 상태 다이어그램
```
idle → uploading → validating → analyzing → completed
                                           ↓
                                         failed (재시도 가능)
```

---

## 흐름 2: 분석 결과 탐색

### 단계

```
1. 분석 결과 뷰어 진입
   → 좌측 패널: 노드 트리 표시
   → 중앙: 선택된 노드/컴포넌트 상세
   → 우측 패널: 속성 및 토큰

2. 디자인 토큰 확인
   → 색상 팔레트 시각화
   → 타이포그래피 스케일 표시
   → 간격/반경 값 목록

3. 컴포넌트 트리 탐색
   → 계층 구조 드릴다운
   → 각 컴포넌트: AI가 제안한 이름 + 신뢰도 점수
   → 이름 수정 가능 (인라인 편집)

4. AI 해석 검토
   → 각 컴포넌트의 해석 근거 확인
   → 동의/수정/거부 처리
   → 신뢰도 낮은 항목 경고 표시
```

---

## 흐름 3: 코드 획득

### 단계

```
1. 코드 뷰어 탭 선택
   → 생성된 파일 목록 표시:
      ├── page.tsx
      ├── components/sections/HeroSection.tsx
      ├── components/cards/FeatureCard.tsx
      └── tokens.json

2. 파일 선택 및 코드 확인
   → 구문 강조(syntax highlighting) 뷰어
   → 읽기 전용 (편집 불가)

3. 코드 추출
   → 파일별 복사 버튼
   → 전체 ZIP 다운로드
```

---

## 선택 이유

### 왜 JSON 붙여넣기를 택했나?
- Figma REST API 연동은 OAuth 토큰 관리 복잡도가 높음
- JSON 복사-붙여넣기는 추가 설정 없이 즉시 시작 가능
- 포트폴리오 데모 환경에서 OAuth 설정 부담 없음

### 왜 패널 기반 UI인가?
- Figma, VS Code처럼 개발자에게 친숙한 패턴
- 좌/중/우 패널 분리로 정보 과부하 방지
- 노드 탐색 + 속성 확인 + 코드 확인을 동시에 가능

---

## 대안 비교

| 접근 방식 | 장점 | 단점 | 선택 |
|----------|------|------|------|
| JSON 붙여넣기 | 간단, 즉시 시작 | 매번 복사 필요 | **선택** |
| Figma REST API | 자동화 | OAuth 복잡도 | 미선택 (Phase 2) |
| 파일 업로드 | 파일 저장 가능 | .json 파일 준비 필요 | **보조 지원** |

---

## 향후 영향

- Figma REST API 연동 시 흐름 1의 업로드 단계를 "URL 입력"으로 대체 가능
- 실시간 동기화 구현 시 분석 결과가 자동 갱신되는 흐름 추가 가능

---

## 화면 흐름 요약

```
/                    (랜딩/대시보드)
  ↓ 새 프로젝트
/projects/new        (프로젝트 생성)
  ↓ 생성 완료
/projects/[id]       (Figma JSON 업로드)
  ↓ 분석 시작
/analysis/[id]       (분석 진행 상태)
  ↓ 완료
/analysis/[id]       (결과 뷰어 — 토큰, 컴포넌트)
  ↓ 코드 탭
/analysis/[id]/code  (코드 뷰어)
```
