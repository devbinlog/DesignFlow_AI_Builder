# 03. 정보 구조 (Information Architecture)

## 목적

이 문서는 DesignFlow AI Builder에서 사용자가 탐색하는 정보의 구조와 계층을 정의한다. 화면 구성, 네비게이션 패턴, 각 화면이 다루는 정보의 범위를 명확히 한다.

---

## 핵심 결정 사항

### 1. 3단계 정보 계층 구조

```
Level 1: 프로젝트 (Project)
  └── Level 2: 분석 실행 (Analysis Run)
        └── Level 3: 분석 결과 항목 (Token / Component / Code File)
```

### 2. 패널 기반 탐색

- 전역 네비게이션: 좌측 사이드바 (프로젝트 목록)
- 로컬 탐색: 좌측 패널 (노드 트리 또는 파일 목록)
- 상세 정보: 우측 패널 (속성, 코드, 피드백)
- 메인 캔버스: 중앙 영역 (핵심 콘텐츠)

---

## 선택 이유

패널 기반 3단 레이아웃을 선택한 이유:
- 분석 도구 특성상 계층적 탐색과 상세 정보 동시 표시 필요
- Figma, Linear, VS Code처럼 개발자에게 친숙한 패턴
- 3단 분리로 좌측(탐색) / 중앙(내용) / 우측(상세) 역할 명확화

---

## 사이트맵

```
DesignFlow AI Builder
├── 대시보드 (/)
│   ├── 최근 프로젝트 목록
│   ├── 새 프로젝트 생성 버튼
│   └── 사용 통계 (분석 횟수 등)
│
├── 프로젝트 목록 (/projects)
│   ├── 프로젝트 카드 목록
│   └── 검색/필터
│
├── 프로젝트 상세 (/projects/[id])
│   ├── 프로젝트 정보
│   ├── Figma JSON 업로드 영역
│   └── 분석 이력 목록
│
└── 분석 결과 뷰어 (/analysis/[id])
    ├── 토큰 탭
    │   ├── 색상 (Colors)
    │   ├── 타이포그래피 (Typography)
    │   ├── 간격 (Spacing)
    │   └── 반경 (Radius)
    ├── 컴포넌트 탭
    │   ├── 컴포넌트 트리
    │   └── 컴포넌트 상세
    └── 코드 탭
        ├── 파일 목록
        └── 코드 뷰어
```

---

## 화면별 정보 구성

### 대시보드 (/)

| 영역 | 정보 |
|------|------|
| 헤더 | 앱 이름, 전역 액션 |
| 최근 프로젝트 | 최근 5개 프로젝트 카드 |
| 빠른 시작 | 새 프로젝트 CTA |

### 분석 결과 뷰어 (/analysis/[id])

```
┌─────────────────────────────────────────────────────────┐
│ Header: 프로젝트명 | 분석 ID | 상태 배지 | 액션          │
├──────────┬──────────────────────────────┬───────────────┤
│          │                              │               │
│ 좌측     │  메인 캔버스                  │  우측         │
│ 패널     │                              │  패널         │
│          │  [토큰 탭]                   │               │
│ 노드     │  색상 팔레트 시각화           │  선택된       │
│ 트리     │  타이포 스케일               │  항목 상세     │
│          │                              │  피드백       │
│          │  [컴포넌트 탭]               │  입력         │
│          │  컴포넌트 계층 + AI 해석     │               │
│          │                              │               │
│          │  [코드 탭]                   │               │
│          │  코드 뷰어 (읽기 전용)       │               │
│          │                              │               │
└──────────┴──────────────────────────────┴───────────────┘
```

---

## 네비게이션 패턴

### 전역 네비게이션
- 좌측 사이드바: 항상 표시 (분석 뷰어 제외 시 접힘 가능)
- 상단 헤더: 브레드크럼 + 전역 액션

### 로컬 네비게이션
- 탭 기반: 토큰 / 컴포넌트 / 코드
- 트리 탐색: 클릭으로 드릴다운
- 뒤로가기: 브레드크럼 클릭

---

## 대안 비교

| 레이아웃 | 장점 | 단점 | 선택 |
|---------|------|------|------|
| 3단 패널 | 탐색+내용+상세 동시 표시 | 좁은 화면 적합성 낮음 | **선택** |
| 단일 스크롤 | 단순함 | 분석 도구에 부적합 | 미선택 |
| 탭 중심 | 집중도 | 컨텍스트 전환 비용 높음 | 미선택 |

---

## 향후 영향

- 분석 뷰어 레이아웃은 향후 컴포넌트 미리보기 패널 추가 시 4단으로 확장 가능
- 노드 트리 구조는 Figma 플러그인 UI와 동일한 패턴 적용 시 코드 재사용 가능
