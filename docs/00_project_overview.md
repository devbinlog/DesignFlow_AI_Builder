# 00. 프로젝트 개요

## 목적

DesignFlow AI Builder는 Figma 디자인 파일의 구조를 분석하여 React + Tailwind CSS 코드를 생성하는 AI 기반 설계-개발 해석 시스템이다. 이 시스템의 목적은 디자인과 개발 사이의 단절(design-dev gap)을 줄이고, 반복적인 수작업 변환 과정을 자동화하여 개발자의 생산성을 높이는 데 있다.

---

## 핵심 결정 사항

### 1. "변환기"가 아닌 "해석 시스템"으로 정의

단순한 Figma-to-Code 변환기(converter)가 아닌, 디자인 의도를 개발 관점에서 해석하는 **interpretation layer**로 정의한다. 이는 단순 변환이 아니라 구조 분석, 패턴 인식, 의미 추론을 포함하기 때문이다.

### 2. AI는 보조 도구

AI(Claude)는 결정의 최종 권한자가 아니다. 모든 AI 출력에는 신뢰도(confidence score)가 포함되며, 사용자가 언제든 수정(override)할 수 있다.

### 3. 포트폴리오 수준의 품질

프로토타입이 아닌 실제 내부 도구처럼 보이는 수준의 구현을 목표로 한다. 에러 처리, 상태 관리, UI/UX 일관성 모두 프로덕션 수준으로 구현한다.

---

## 선택 이유

| 결정 | 이유 |
|------|------|
| React + Tailwind 코드 생성 타겟 | 현재 가장 널리 사용되는 조합. 포트폴리오 타깃층에 최적 |
| FastAPI 백엔드 | 비동기 처리, 빠른 개발, Pydantic 기반 타입 안전성 |
| Claude AI | 구조화 JSON 출력 품질이 우수하고, API 안정성이 높음 |
| PostgreSQL JSONB | 반구조화 데이터(Figma JSON) 처리에 최적화 |
| 패널 기반 UI | 디자인 도구(Figma, Figma DevMode)와 유사한 UX 패턴 |

## 대안 비교

### AI 모델 선택
| 모델 | 장점 | 단점 | 선택 |
|------|------|------|------|
| Claude Sonnet 4.6 | 구조화 출력 우수, 빠른 응답 | - | **선택** |
| GPT-4o | 범용성 | 구조화 JSON 일관성 낮음 | 미선택 |
| Gemini Pro | 멀티모달 | 코드 생성 일관성 낮음 | 미선택 |

### 코드 생성 타겟
| 타겟 | 장점 | 단점 | 선택 |
|------|------|------|------|
| React + Tailwind | 범용, 포트폴리오 활용도 높음 | - | **선택** |
| React + CSS Modules | 스코핑 명확 | 토큰 연동 복잡 | 미선택 |
| Vue + UnoCSS | 경량 | 사용층 좁음 | 미선택 |

## 향후 영향

- 이 시스템의 분석 파이프라인은 향후 Figma 플러그인 방식으로 확장 가능
- AI 프롬프트 버전 관리를 통해 모델 교체 시 품질 회귀 테스트 가능
- JSONB 기반 저장 구조는 향후 다른 디자인 도구(Sketch, XD) 지원 시 확장 용이
