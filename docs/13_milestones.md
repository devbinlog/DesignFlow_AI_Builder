# 13. 마일스톤

## 목적

이 문서는 DesignFlow AI Builder의 개발 단계와 각 마일스톤의 목표를 정의한다. 각 단계의 완료 기준과 산출물을 명확히 하여 진행 상황을 추적한다.

---

## 핵심 결정 사항

### 단계별 구분 기준
- M0: 기반 구조 (문서 + 인프라)
- M1: 백엔드 파이프라인
- M2: 프론트엔드 UI
- M3: 시스템 통합 및 완성

---

## 선택 이유

4단계 마일스톤으로 구분한 이유:
- 각 단계가 독립적으로 검증 가능
- 백엔드 완성 후 프론트엔드 시작 → 의존성 명확화
- 포트폴리오 발표 시 진행 단계 설명 용이

---

## 향후 영향

마일스톤이 명확하므로 포트폴리오 소개 시 "어느 단계까지 구현했는가"를 명확히 설명 가능.

## M1: 백엔드 파이프라인 (Backend Pipeline)

### 목표
- FastAPI 서버 구축
- Figma JSON 파서 구현
- AI 파이프라인 구현
- DB 연동

### 완료 기준
- [ ] `GET /api/v1/health` → 200 응답
- [ ] `POST /api/v1/projects` → 프로젝트 생성 및 DB 저장
- [ ] `POST /api/v1/analysis` → 파이프라인 실행
  - [ ] Figma JSON 파싱 성공
  - [ ] 디자인 토큰 추출 (색상, 타이포, 간격)
  - [ ] AI 해석 결과 반환 (신뢰도 포함)
  - [ ] 코드 생성 결과 반환
- [ ] `GET /api/v1/analysis/{id}` → 결과 조회
- [ ] 에러 케이스 처리 (잘못된 JSON, AI 실패)

### 산출물
- 동작하는 FastAPI 서버
- 완성된 분석 파이프라인
- Swagger UI 문서 (`/docs`)
